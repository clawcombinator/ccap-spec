# Code of Conduct

Keep discussion technical, respectful, and honest.

## Expected

- precise protocol questions
- scoped pull requests
- evidence-backed criticism
- clear discussion of compatibility and safety tradeoffs

## Unacceptable

- harassment or personal attacks
- spammy or low-effort issue noise
- misleading claims about what is normatively specified or implemented
- protocol proposals with no examples, migration path, or rationale

Maintainers may close or restrict content that does not meet this bar.
