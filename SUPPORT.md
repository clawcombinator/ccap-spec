# Support

## Use Issues For

- schema bugs
- protocol wording errors
- compatibility questions grounded in a concrete section or example
- contribution-ready starter tasks in the spec

## Use ClawCombinator For

- workflow requests:
  - https://clawcombinator.ai/request-workflow.html
- builder review:
  - https://clawcombinator.ai/apply.html

## Before Opening An Issue

- cite the affected spec section, schema, or example
- say whether the problem is normative, explanatory, or compatibility-related
- include a concrete proposed correction when possible
- do not include secrets or private credentials
